# PATH — 最短経路問題の実体

実装している planning graph の明示。Dijkstra を階層化した contraction hierarchy 風の構造。

## 階層は 4 レベル

| Level | 抽象度 | ノードの例 | 誰が判断 |
|-------|-------|----------|---------|
| **L0** | 世界観 | 「情報信頼が権威と貨幣から切り離された世界」 | あなた |
| **L1** | 戦略マイルストーン | M1〜M7（N=1価値 → 10人 → 100人 → 1000人） | 僕（ビジョン変更はあなた） |
| **L2** | 小経路（マイルストーン内の選択肢） | M1-a / M1-b / M1-c など | 僕 |
| **L3** | 戦術アクション | `cli/timeline.js` を書く、PR送る | 僕 |

## 現在位置（2026-04-15）

```
L0:  情報信頼の分離        [到達は長期、常時フラグとして保持]
L1:  S0 → M1 エッジ上      ◀ 現在ここ
L2:  M1-a (個人タイムライン可視化)  ◀ 選択済み
```

## Planning graph（メインチャネル + ショートカット）

メインチャネル：
```
S0 ──▶ M1 ──▶ M2 ──▶ M3 ──▶ M4 ──▶ M5 ──▶ M6 ──▶ M7
```

各エッジのコストと確率：

| Edge | cost | prob | 備考 |
|------|------|------|------|
| S0→M1 | 僕の2-4h | 0.9 | M1-a実装 |
| M1→M2 | あなたのXポスト + 依存 | 0.2-0.3 | 離散ヒット依存 |
| M2→M3 | 待ち+微調整 | 0.6 | 自然発生 |
| M3→M4 | 待ち | 0.2-0.4 | 偶然性高 |
| M4→M5 | 待ち or 僕のPR | 0.3 | 人気ツールに組込要 |
| M5→M6 | 網内成長 | 0.8 | networkeffect |
| M6→M7 | 時間 | 0.9 | 臨界点自然拡大 |

Contraction Hierarchies ショートカット：

| Edge | cost | prob | 期待ジャンプ |
|------|------|------|------------|
| S0 ⇢ M4 | あなたのHN Show HN | 0.05-0.10 | M1+M2+M3を飛ばす |
| S0 ⇢ M3 | influencer DM | 0.02-0.05 | 人選要 |
| M1 ⇢ M5 | 直接 integration PR | 0.10-0.20 | 対象選定要 |
| M4 ⇢ M6 | mediaカスケード | 0.10 | M4達成依存 |

## Dijkstra 風の重み付き到達可能性

現在までの推定 g-score（S0からの期待コスト）：

| Node | g-score (期待時間) | 先行エッジ | 状態 |
|------|------------------|----------|------|
| S0 | 0 | — | 現在 |
| M1 | 3h | S0→M1 via M1-a | 進行中 |
| M2 | 3h + 離散待機 | M1→M2 | blocked-on-M1 |
| M3 | +1-3d | M2→M3 | blocked-on-M2 |
| M4 | +2-7d | M3→M4 or shortcut | blocked |
| M5 | +1-4w | M4→M5 or M1→M5 shortcut | blocked |
| M6 | +2-8w | M5→M6 | blocked |
| M7 | +1-6m | M6→M7 | blocked |

## Pruned edges（枝刈り済 = 実行しない）

- お金・権威・普通の営業 → coverage ∞（禁則）
- プロダクトUI制作 → ∞（禁則）
- あなたの認知を使うvision以外の質問 → ∞（禁則）
- 資格情報なしのアカウント投稿（HN/X/Reddit） → ∞（僕には閉じている）

## Relax ルール

新規情報取得時、以下の edges を re-estimate：

- あなたの発言 → heuristic更新（探索/活用比率変更）
- chain event（新規参加者） → M2/M3の距離を再計算
- 外部PR merge → M4/M5 距離を短縮
- 時間経過（鮮度低下） → 全edgesのprobをdecay

## ポリシー

1. 最小の g-score を持つ未展開ノードから拡張（Dijkstra素朴）
2. ただし**ショートカット edge の期待値 > 線形路の期待値 × 3** なら ショートカットに賭ける
3. blocked-on-M1 のものは M1 達成まで優先度下限まで下げる

## 2026-04-15 14:00頃の判断記録

**事象**：X投稿後30分で view=0 likes=0 / Reddit setup詰まり。
**分析**：フォロワー基盤なし→X単独amplifyのEV低下。Reddit新規アカウントもbot判定リスク。

## 2026-04-15 17:00頃の誤り診断

**観察**：ユーザーから「X運用できんじゃない？」と指摘。
**診断**：僕は X を「self-post」の1ノードで扱っていた。reply / quote-RT / DM を**明示的にenumerateせず**、親ノード「X」の posterior に集約してしまった。結果、self-post失敗の観測で reply sub-edge も同時に切ってしまった。

**根本原因**：
1. 抽象粒度の誤り（edge 定義が粗すぎ）
2. 粒度を誤った Bayes 更新（親に逆伝播）
3. 枝刈り後の探索停止
4. action space enumerate の不完全

**修正**：
- 全ての edge を末端粒度で定義
- 親 prune 前に子 enumerate 必須化
- posterior は最深粒度のみ更新
- N アクションごとに graph audit

## アルゴリズム不変条件（invariant）

以下を破った時点で最適化は歪む：

- **I1**: edge 定義は観測可能な独立 action に1対1対応
- **I2**: Bayes 更新は観測が発生した edge **のみ**に適用
- **I3**: 親 edge の prune は、全ての子 edge の観測 posterior が閾値未満の時のみ
- **I4**: 各サイクルで1つ以上の新しい edge を action space に追加可能か検査
- **I5**: プロダクト機能追加は「大問い→仮説→実装」経路を辿るもののみ。トレンド追従・他社模倣は却下
- **I6**: OZC は大問いへの1つの仮説実装にすぎない。単独での結論主張は禁止
